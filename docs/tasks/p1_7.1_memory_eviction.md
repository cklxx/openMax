## Goal

Implement age + relevance based eviction for workspace memory. When the memory store exceeds a configurable limit, the least relevant entries (scored by age and access frequency) are automatically evicted.

## Key Files

- `src/openmax/memory/store.py`
- `src/openmax/memory/_utils.py`

## Acceptance Criteria

- [ ] Memory entries track `last_accessed` timestamp, updated on each read
- [ ] Configurable `max_entries` limit (default 100)
- [ ] Eviction policy scores entries by age and access frequency
- [ ] Least relevant entries are evicted when limit is exceeded
- [ ] Eviction runs automatically on new entry insertion
- [ ] Tests cover: eviction trigger, scoring logic, last_accessed updates, boundary conditions

## Implementation Notes

- Add `last_accessed: float` field to `MemoryEntry` in `models.py` (or wherever the dataclass lives)
- Add `MAX_MEMORY_ENTRIES` constant to `_utils.py`
- Eviction score formula: combine age (time since creation) and staleness (time since last access) with configurable weights
- Evict entries with lowest relevance score when count exceeds limit
- Update `store.py` read paths to bump `last_accessed`
- Update `store.py` write path to trigger eviction check after insert
