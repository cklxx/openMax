# 11.1 Branch Isolation & Auto-Merge

- **Priority**: P1
- **Status**: done
- **Dependencies**: —

## Goal

Each dispatched agent works on its own git branch, preventing file-level conflicts between parallel agents. On task completion, the lead agent can merge the agent's branch back.

## Acceptance Criteria

- [x] `dispatch_agent` creates a unique branch per agent (`openmax/<task-id>`)
- [x] Agent work is isolated to its branch
- [x] `merge_agent_branch` tool merges completed agent branches back to the base branch
- [x] Merge conflicts are detected and reported to the lead agent
- [x] Tests cover branch creation, merge success, and conflict detection

## Key Files

- `src/openmax/lead_agent/tools.py` — `dispatch_agent` branch creation (line ~542), `merge_agent_branch` tool (line ~860)
- `src/openmax/lead_agent/prompts/lead_agent.md` — agent dispatch guidance
- `tests/test_lead_agent.py` — branch isolation tests (line ~1504, ~1553)

## Implementation Notes

Branch isolation is handled in `dispatch_agent` by creating a new branch before the agent starts work. The `merge_agent_branch` tool performs a `git merge` and returns structured results including conflict detection.
