# 11.2 Context Compression Protocol

**Status:** done
**Priority:** P2

## Goal

Improve sub-agent context by compressing and injecting relevant information at dispatch time, respecting a configurable token budget.

## Acceptance Criteria

- [x] `dispatch_agent` accepts optional `context_budget_tokens` parameter (default 2000)
- [x] `_compress_context()` helper compresses text to fit within token budget
- [x] Compression keeps first paragraph and key bullet points
- [x] Workspace memory context is compressed before injection
- [x] Tests verify budget is respected and long context gets compressed

## Implementation

- Added `_compress_context(context, budget)` in `tools.py` — uses `len//4` token estimate, preserves first paragraph + bullet lines within budget
- Added `context_budget_tokens` parameter to `dispatch_agent` schema
- Applied compression to the workspace memory injection path
- Added 3 tests: passthrough under budget, truncation over budget, dispatch integration
