# Task 10.1: Failure Auto-Retry

| Field | Value |
|-------|-------|
| **Priority** | P1 |
| **Dependencies** | — |

## Goal

Detect agent failures and auto-retry up to 2 times. When an agent pane exits with an error or matches known failure patterns, the system automatically retries the task before reporting a permanent failure.

## Key Files

- `src/openmax/lead_agent/tools.py`
- `src/openmax/lead_agent/types.py`
- `src/openmax/pane_manager.py`
- `docs/lead_agent.md`

## Acceptance Criteria

- [ ] Track `retry_count` per pane/task
- [ ] Detect failure via exit code or error patterns in pane output
- [ ] Auto-retry failed tasks (max 2 retries)
- [ ] Report permanent failures after max retries exhausted
- [ ] Lead agent system prompt updated with retry guidance
- [ ] Tests cover: retry trigger, max retry limit, permanent failure reporting, retry count tracking

## Implementation Notes

- Add `retry_count: int = 0` and `max_retries: int = 2` fields to the subtask/pane tracking structure in `types.py`
- In `tools.py`, detect failure when `read_pane_output` or `report_completion` encounters error signals (non-zero exit, Traceback, FAILED patterns)
- On failure detection, if `retry_count < max_retries`: kill the failed pane, re-dispatch the same task, increment `retry_count`
- On permanent failure (`retry_count >= max_retries`): report with all retry context
- Update `lead_agent.md` to instruct the lead agent on retry behavior and when to intervene manually
- Consider adding retry metadata to session events for observability
