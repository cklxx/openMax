## Goal

Validate phase transitions via the `transition_phase` tool. Ensure the lead agent follows the defined phase sequence (plan → dispatch → monitor → finish) with proper validation and observability.

## Acceptance Criteria

- [ ] `transition_phase` tool validates that `from_phase` matches the current session phase
- [ ] Invalid transitions are rejected with a clear error message
- [ ] Phase anchor events are recorded in session state
- [ ] Dashboard displays the current phase
- [ ] Lead agent system prompt updated with phase transition guidance
- [ ] Tests cover: valid transitions, invalid from_phase rejection, phase event recording, dashboard display

## Implementation Notes

- Add `current_phase: str = "plan"` to `LeadAgentRuntime` or session state
- `transition_phase` tool: accept `from_phase` and `to_phase` args, validate `from_phase == current_phase`, validate `to_phase` is a valid next phase, update state, record event
- Add `"transition_phase"` to `_TOOL_CATEGORIES` in `formatting.py` and add format block in `_format_tool_use`
- Update `dashboard.py` to show current phase in the status display
- Update `lead_agent.md` with phase transition rules and guidance
- Consider allowing `monitor` → `dispatch` for re-dispatch scenarios
